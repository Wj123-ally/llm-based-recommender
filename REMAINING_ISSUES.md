## 核心问题分析

### 为什么排除逻辑没有生效？

从测试结果分析：

1. **第一轮** (`new_request`):
   - 推荐: `['a7ed7dbdc561889f', 'a41ed4b1db75130d', '3fbc87a0c58279ae']`
   - `recommended_product_ids` = 这3个ID

2. **第二轮** (`clarification`): "300-500元"
   - `context_mode` = "clarification" ✓
   - `recommended_ids` = 3个ID ✓
   - **应该触发排除逻辑**
   - 但结果: 仍推荐相同的3个商品 ❌

3. **第三轮** (`follow_up`): "有耐克/阿迪达斯吗"
   - `context_mode` = "follow_up" ✓
   - `recommended_ids` = 3个ID ✓  
   - **应该触发排除逻辑**
   - 但结果: 仍推荐相同的3个商品 ❌

### 可能的原因

#### 原因1: 排除逻辑触发了，但数据库商品太少
如果数据库中符合"夏季透气运动鞋"条件的商品只有这3个，那么：
- 排除这3个后 → 候选为0
- `retrieve_products()` 返回空列表
- 但代码中没有处理这种情况的回退逻辑

#### 原因2: 排除逻辑没有触发
可能是因为：
- 日志没有输出（日志级别问题）
- `current_query_has_new_product` 判断有问题
- State传递有问题

#### 原因3: 检索本身的问题
即使不排除，filters失效后的语义检索也应该根据新的query返回不同结果：
- 第一轮: "夏季透气运动鞋"
- 第二轮: "夏季透气运动鞋 300-500元"  
- 第三轮: "耐克 跑步鞋"

但向量相似度可能导致返回相同的top-3。
